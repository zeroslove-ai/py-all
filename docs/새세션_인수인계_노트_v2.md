## 7. 실수 방지 체크리스트

- [ ] Storage 파일명은 **실제 조회 후** 매칭 (사용자 키값 ≠ 실제 파일명)
- [ ] 하위 폴더(`sex/`)는 **별도 prefix**로 재조회
- [ ] YML 무결성: nodes/edges 수, dangling, 중복 반드시 검증
- [ ] Assigner 노드: `items` 배열 + `version: "2"` (variables 아님)
- [ ] if-else `case_id`: `"true"` (문자열)
- [ ] 새 변수는 `conversation_variables`에도 선언
- [ ] `app.name` = 파일명과 동일 (`게임빌더_v64`)
- [ ] **새 버전 생성 시 사용자에게 "Supabase에도 저장할까요?" 물어보고 동의 후만 INSERT** (현재 정책: 저장 안 함)
