## 적용 기준

- 원본 `image_library` 행과 Storage 파일은 삭제하지 않았다.
- `image_url` 경로에 `/sex/`가 있으면 `image_pool=sex`, 나머지는 `general`로 분류했다.
- DB `character_id`를 절대 기준으로 사용하고 파일명·설명의 다른 캐릭터 이름은 재분류 근거로 쓰지 않았다.
- 캐릭터별 최대 `general 15 + sex 20`으로 선별했다.
- 장면 유형을 라운드로빈해 같은 자세·표정에 과도하게 몰리지 않도록 했다.
- 태그와 짧은 설명은 기존 `situation`과 파일 경로 키워드를 정규화해 생성했다.

## 표시 규칙

- 성행위 장면은 sex 풀을 우선하며 일반 특수 이미지로 대체하지 않는다.
- 암시 활성화 또는 실제 적용된 최면깊이 증가 시 `hypnosis_onset`을 우선 표시한다.
- 호감도 70 최초 도달 또는 최면깊이·순응도 70 동시 최초 도달 시 `heart_eyes`를 우선 표시한다.
- 임계값 도달 이후 매 턴 특수 이미지를 강제하지 않고 일반 장면 매칭으로 돌아간다.
- Commit에서 확정된 이미지 ID를 `last_image_id`로 저장하며, 프론트와 `/api/image`는 저장된 ID를 사용한다.
